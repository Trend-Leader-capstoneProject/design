
1.  회원가입 직후 자동 로그인으로 토큰을 발급할지, 
아니면 카테고리 목록 조회만 인증 불필요로 열어둘지

2.  전체 교체 방식과 변경분 반영 방식 중 무엇을 택할지

| 방식     | 설명                         | 장점                  | 단점                     | 추천       |
| ------ | -------------------------- | ------------------- | ---------------------- | -------- |
| 전체 교체  | 기존 관심사를 모두 삭제 후 새 목록을 재등록  | 구현이 단순하고 버그 가능성이 낮음 | `created_at`이 모두 새로 바뀜 | MVP 추천   |
| 변경분 반영 | 기존 목록과 새 목록을 비교해 추가/삭제만 처리 | 데이터 변화 추적에 유리       | 로직이 조금 복잡함             | 확장 단계 추천 |

3. 추천 알고리즘을 처음부터 복잡하게 만들지 않을 것,
    MVP에서는 사용자 관심 카테고리 → 트렌드 카테고리 매핑 → 최신 순위/점수 정렬 정도로 시작하는 게 가장 안정적임.
