지금 상태에서는 **화면을 더 많이 만드는 것보다 먼저 “화면이 왜 필요한지”를 정리해야 하는 단계**입니다.
무대뽀로 만들기 시작하면 화면 수는 늘어나는데, 나중에 발표할 때 “이 화면이 왜 필요한가요?”라는 질문에 답하기 어려워집니다.

현재 올려주신 화면을 보면 이미 꽤 많은 요소가 있습니다.

```text
로그인
회원가입
관심사 선택
관심사 기반 메인
검색
검색 결과 / 상세
사이드 메뉴
프로필
내 글 목록
댓글 목록
비밀번호 변경
탈퇴 / 로그아웃 확인 모달
```

문제는 화면이 부족하다기보다, 아직 **사용자 흐름 기준으로 묶이지 않아서 머릿속에 전체 앱 구조가 잘 안 보이는 상태**에 가깝습니다.

# 1. 먼저 “화면”이 아니라 “사용자 행동”부터 잡기

Trend Leader의 핵심 사용자는 이런 사람입니다.

> 내가 관심 있는 분야의 최신 트렌드를 빠르게 보고, 필요하면 자세히 확인하거나 저장하고 싶은 사용자

그러면 앱의 핵심 행동은 5개로 정리됩니다.

```text
1. 관심사 선택하기
2. 내 관심사 기반 트렌드 보기
3. 트렌드 상세 내용 확인하기
4. 직접 검색하기
5. 저장하거나 나중에 다시 보기
```

여기서부터 화면이 나와야 합니다.

즉, 출발점은 이렇게 잡으면 됩니다.

```text
사용자 행동 → 필요한 화면 → 필요한 UI 요소 → 데이터 저장 구조
```

반대로 하면 위험합니다.

```text
화면 먼저 만들기 → 기능을 억지로 끼워넣기 → 흐름이 꼬임
```

---

# 2. 현재 화면을 기준으로 앱 구조 다시 묶기

지금 만든 화면을 기능별로 다시 묶으면 이렇게 됩니다.

```text
[인증 영역]
- 로그인
- 회원가입
- 비밀번호 변경

[초기 설정 영역]
- 관심사 선택

[트렌드 탐색 영역]
- 메인 홈
- 관심사별 트렌드 리스트
- 검색 화면
- 트렌드 상세 화면

[사용자 활동 영역]
- 저장한 트렌드
- 내 글 목록
- 댓글 목록

[마이페이지 영역]
- 프로필
- 내 정보
- 관심사 재설정
- 로그아웃
- 회원 탈퇴
```

여기서 현재 가장 약한 부분은 **트렌드 탐색 영역**입니다.
캡스톤 기획의 핵심은 관심사 기반 트렌드 앱인데, 지금은 마이페이지/계정 관련 화면이 꽤 있고, 정작 “트렌드를 어떻게 탐색하는가” 화면이 더 확장될 필요가 있습니다.

---

# 3. 화면 확장은 이 순서로 하는 것이 좋습니다

화면을 많이 추가해야 할 때는 아래 순서로 확장하면 덜 막힙니다.

```text
1단계: 전체 앱 지도 만들기
2단계: 핵심 사용자 흐름 만들기
3단계: 화면 목록 만들기
4단계: 각 화면의 목적 한 줄 정의
5단계: 저충실도 와이어프레임 작성
6단계: 필요한 화면만 디자인 고도화
```

가장 먼저 해야 할 것은 Figma에서 예쁜 화면을 추가하는 게 아니라, 아래 같은 **앱 지도**를 만드는 것입니다.

```text
앱 실행
  ↓
로그인 / 회원가입
  ↓
관심사 선택
  ↓
내 트렌드 홈
  ├─ 관심사별 트렌드 리스트
  │    └─ 트렌드 상세
  ├─ 최신 이슈
  │    └─ 이슈 상세
  ├─ 검색
  │    └─ 검색 결과
  │          └─ 트렌드 상세
  ├─ 저장한 트렌드
  │    └─ 저장한 트렌드 상세
  └─ 마이페이지
       ├─ 프로필 수정
       ├─ 관심사 재설정
       ├─ 내 글 목록
       ├─ 댓글 목록
       ├─ 비밀번호 변경
       └─ 로그아웃 / 회원탈퇴
```

이렇게 먼저 그리면 “내가 어떤 화면을 더 만들어야 하지?”가 훨씬 명확해집니다.

# 4. 지금 추가하면 좋은 화면 목록

현재 기획 기준으로는 아래 화면들을 추가하면 구성이 더 탄탄해집니다.

## 우선순위 높음

```text
1. 내 트렌드 홈 개선 화면
2. 카테고리별 트렌드 리스트 화면
3. 트렌드 상세 분석 화면
4. 최신 이슈 탭 화면
5. 저장한 트렌드 화면
6. 관심사 재설정 화면
```

## 우선순위 중간

```text
7. 알림 설정 화면
8. 검색 결과 없음 화면
9. 로딩 화면
10. 서버 오류 화면
11. 트렌드 데이터 없음 화면
```

## 후순위

```text
12. 커뮤니티 게시글 목록
13. 게시글 작성
14. 게시글 상세
15. 댓글 작성
```

중요한 점은 **정상 화면만 만들면 부족하다**는 것입니다.
앱 화면 구성에서는 정상 화면보다 아래 상태들이 더 기획력을 보여줍니다.

```text
로딩 중
데이터 없음
검색 결과 없음
에러 발생
로그인 필요
관심사 미선택
```

발표에서 이런 화면까지 보여주면 “실제 앱 흐름을 고려했다”는 인상을 줄 수 있습니다.

# 5. 추천 하단 탭 구조

현재 하단 아이콘이 조금 추상적으로 보입니다. 발표용이라면 탭 이름이 명확한 편이 좋습니다.

추천 구조는 이겁니다.

```text
[내 트렌드] [최신 이슈] [검색] [저장] [마이]
```

다만 모바일 화면에서 5개 탭은 조금 많을 수 있습니다. 그래서 MVP 기준으로는 4개가 더 안정적입니다.

```text
[내 트렌드] [최신 이슈] [저장] [마이]
```

검색은 상단 검색창으로 넣으면 됩니다.

```text
┌────────────────────┐
│ 🔍 트렌드 검색하기   │
└────────────────────┘
```

# 6. 핵심 화면 와이어프레임

## 6-1. 내 트렌드 홈

현재 메인 화면은 방향이 좋습니다. 다만 “오늘의 추천”, “카테고리”, “트렌드 리스트”가 더 명확히 나뉘면 좋습니다.

```text
┌────────────────────────────┐
│ ⋮        TL             🔔 │
├────────────────────────────┤
│ 오늘님을 위한              │
│ 오늘의 맞춤 트렌드          │
│ 관심 분야 기반 추천         │
│                            │
│ 🔍 관심 키워드 검색          │
│                            │
│ 관심 카테고리               │
│ [패션] [IT/지식] [엔터]     │
│                            │
│ 🔥 오늘의 추천 트렌드        │
│ ┌────────────────────────┐ │
│ │ 1. 내 관심사 급상승     │ │
│ │ 음식 · 오늘 상승        │ │

이 화면이 앱의 차별화 포인트가 되어야 합니다.
단순 키워드 설명이 아니라 “왜 뜨는지”를 보여줘야 합니다.

AI 요약과 관련 키워드를 통해 해당 트렌드가 왜 주목받는지 이해할 수 있다.
```

# 7. 화면을 더 만들 때 쓰는 체크리스트

앞으로 새 화면을 만들 때마다 아래 5개를 먼저 적고 시작하세요.

```text
1. 이 화면의 목적은 무엇인가?
2. 사용자가 이 화면에서 누를 수 있는 것은 무엇인가?
3. 이전 화면은 무엇인가?
4. 다음 화면은 무엇인가?
5. 데이터가 없거나 오류가 나면 어떻게 보여줄 것인가?
```

예를 들어 트렌드 상세 화면이면 이렇게 됩니다.

```text
화면명: 트렌드 상세 화면

목적:
사용자가 특정 트렌드의 의미, 상승 이유, 관련 키워드를 확인한다.

이전 화면:
내 트렌드 홈, 최신 이슈 탭, 검색 결과 화면

주요 동작:
저장하기
관련 키워드 누르기
뒤로가기

다음 화면:
저장한 트렌드 화면
관련 키워드 검색 결과 화면

예외 상태:
AI 요약 생성 실패
관련 키워드 없음
트렌드 데이터 삭제됨
```

이렇게 적고 나서 Figma를 그리면 훨씬 덜 막힙니다.

# 8. 지금 당장 하면 좋은 작업 순서

현재 상황에서는 이렇게 진행하는 걸 추천합니다.

```text
1. Figma에서 새 페이지 생성: "IA / Flow"
2. 앱 전체 흐름도를 네모 박스로 먼저 그림
3. 화면을 4개 그룹으로 나눔
   - 인증
   - 온보딩
   - 트렌드 탐색
   - 마이페이지
4. 각 그룹마다 필요한 화면 목록 작성
5. 핵심 화면 6개만 저충실도 와이어프레임으로 복제
6. 이후 색상/아이콘/카드 디자인 적용
```

Figma에서는 처음부터 예쁜 화면을 만들지 말고, 아래처럼 회색 박스로만 먼저 잡아도 됩니다.

```text
Header
Search
Category Chips
Trend Card List
Bottom Navigation
```

즉, 지금은 디자인 단계라기보다 **화면 설계 단계**입니다.

# 9. 현재 화면에서 바로 보완하면 좋은 점

지금 만든 화면을 기준으로 보면 보완 방향은 이렇습니다.

```text
1. 하단 탭 아이콘에 텍스트 라벨 추가
2. 메인 화면의 목적을 “내 트렌드 홈”으로 명확히 변경
3. 검색 화면과 트렌드 리스트 화면의 역할 분리
4. 트렌드 상세 화면에 AI 요약 / 상승 이유 / 관련 키워드 추가
5. 저장한 트렌드 화면 추가
6. 최신 이슈 탭 화면 추가
7. 관심사 재설정 화면 추가
8. 데이터 없음 / 검색 결과 없음 / 오류 화면 추가
```

특히 하단 탭은 아이콘만 있으면 발표 때 설명이 필요해집니다.
아래처럼 라벨을 붙이는 편이 좋습니다.

```text
🔥 내트렌드
▣ 최신
🔍 검색
☆ 저장
👤 마이
```

또는 4개 탭이면:

```text
🔥 내트렌드
▣ 최신이슈
☆ 저장
👤 마이
```

# 10. 핵심 정리

지금은 화면을 무작정 늘릴 단계가 아니라, **앱의 흐름을 먼저 고정해야 하는 단계**입니다.

가장 추천하는 출발 방식은 이것입니다.

```text
사용자 행동 5개 정의
→ 필요한 화면 목록 작성
→ 화면별 목적 한 줄 작성
→ 정상/빈 상태/오류 상태 구분
→ 저충실도 와이어프레임 제작
→ 디자인 적용
```

현재 Trend Leader 기준으로 가장 먼저 추가하면 좋은 화면은 아래 6개입니다.

```text
1. 내 트렌드 홈 개선
2. 카테고리별 트렌드 리스트
3. 트렌드 상세 분석
4. 최신 이슈 탭
5. 저장한 트렌드
6. 관심사 재설정
```

이렇게 출발하면 “화면을 더 만들어야 한다”는 막막함이
**“사용자 흐름에 필요한 화면을 하나씩 채운다”**는 작업으로 바뀝니다.
