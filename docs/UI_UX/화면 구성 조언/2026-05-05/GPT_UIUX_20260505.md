업로드해주신 기획 수정안을 보면 핵심 방향이 **“최신 트렌드 전체 제공” → “내 관심사 기준으로 최신 흐름을 골라보는 개인화 트렌드 앱”**으로 정리되었습니다. 특히 기획 수정안에서는 대상 타깃을 “패션, 뷰티, 음식, 게임 등 자신의 관심 분야 최신 트렌드만 골라 보고 싶은 10~20대 사용자”로 좁히고, 핵심 기능을 **관심사 기반 트렌드 탭**, 보조 기능을 **최신 이슈 탭**으로 나눈 점이 중요합니다.

이 기준이면 화면은 단순히 “랭킹 리스트”만 늘리는 것보다, 아래처럼 **관심사 선택 → 개인화 홈 → 트렌드 상세 분석 → 최신 이슈 탐색 → 저장/마이페이지** 흐름으로 확장하는 것이 좋습니다.

## 1. 전체 화면 구조 제안

현재 기획을 기준으로 하면 앱의 주요 화면은 이렇게 나누는 것이 자연스럽습니다.

```text
1. 스플래시 / 시작 화면
2. 로그인 / 회원가입 화면
3. 관심 카테고리 선택 화면
4. 메인 홈 화면
5. 관심사 기반 트렌드 탭
6. 트렌드 상세 화면
7. 최신 이슈 탭
8. 검색 / 필터 화면
9. 저장한 트렌드 화면
10. 마이페이지 / 관심사 재설정 화면
```

다만 캡스톤 MVP 기준으로는 너무 많은 화면을 한 번에 구현하기보다, 우선순위를 나누는 것이 좋습니다.

### MVP 우선 구현 화면

```text
1. 로그인 / 회원가입
2. 관심 카테고리 선택
3. 메인 홈
4. 관심사 기반 트렌드 리스트
5. 트렌드 상세
6. 최신 이슈 탭
7. 마이페이지
```

### 후순위 확장 화면

```text
1. 저장한 트렌드
2. 알림 설정
3. 검색 기록
4. 관심 키워드 직접 등록
5. AI 챗봇형 트렌드 질문
6. 커뮤니티 / 의견 공유
```

# 2. 추천 앱 구조

기획안의 핵심은 **관심사 기반 개인화**이므로, 하단 탭은 너무 많지 않게 4개 정도가 적당합니다.

```text
[홈] [최신이슈] [저장] [마이]
```

또는 조금 더 기획 의도를 강조하면:

```text
[내 트렌드] [전체 이슈] [저장] [마이]
```

개인적으로는 발표와 기획 설명에서 더 명확한 이름은 아래입니다.

```text
[내 트렌드] [최신 이슈] [저장] [마이]
```

이렇게 하면 사용자가 앱에 들어오자마자 “이 앱은 전체 뉴스 앱이 아니라 내 관심사 중심 앱이구나”라고 이해하기 쉽습니다.

# 3. 화면별 확장 설계

## 3-1. 온보딩 / 관심사 선택 화면

처음 실행 시 사용자의 관심 카테고리를 선택하게 합니다. 기획안의 “처음 실행 시 패션, 뷰티, 게임, 음식 등 관심 카테고리 선택” 흐름과 직접 연결됩니다.

```text
┌────────────────────────────┐
│ Trend Leader               │
│                            │
│ 어떤 트렌드를 보고 싶나요? │
│ 관심 있는 분야를 선택해요   │
│                            │
│ ┌──────┐ ┌──────┐ ┌──────┐ │
│ │ 패션 │ │ 뷰티 │ │ 게임 │ │
│ └──────┘ └──────┘ └──────┘ │
│                            │
│ ┌──────┐ ┌──────┐ ┌──────┐ │
│ │ 음식 │ │ 음악 │ │ 여행 │ │
│ └──────┘ └──────┘ └──────┘ │
│                            │
│ ┌──────┐ ┌──────┐ ┌──────┐ │
│ │ IT   │ │ 영화 │ │ 운동 │ │
│ └──────┘ └──────┘ └──────┘ │
│                            │
│        [선택 완료]          │
└────────────────────────────┘
```

### 배치 포인트

카테고리는 단순 체크박스보다 **둥근 카드형 버튼**으로 두는 것이 좋습니다.

예시:

```text
[👗 패션] [💄 뷰티] [🎮 게임]
[🍜 음식] [🎧 음악] [✈️ 여행]
```

이 화면에서 중요한 것은 “많이 선택할수록 좋아요”가 아니라, **최소 3개 선택** 같은 조건을 주는 것입니다.

```text
관심 분야를 3개 이상 선택해주세요.
선택한 분야를 기준으로 메인 화면이 구성됩니다.
```

## 3-2. 메인 홈 화면

홈은 앱의 첫인상을 결정합니다. 여기서는 사용자가 선택한 관심사를 기준으로 “오늘의 내 트렌드”를 보여주는 것이 좋습니다.

```text
┌────────────────────────────┐
│ Trend Leader        🔔 ⚙   │
│                            │
│ 한재님을 위한 오늘의 트렌드 │
│ 관심사 기반으로 골라봤어요 │
│                            │
│ ┌────────────────────────┐ │
│ │ 오늘의 추천 키워드      │ │
│ │ #무신사 #탕후루 #로블록스│ │
│ └────────────────────────┘ │
│                            │
│ 관심 카테고리              │
│ [패션] [게임] [음식] [뷰티] │
│                            │
│ 🔥 패션 트렌드             │
│ ┌────────────────────────┐ │
│ │ 1. 여름 린넨 셔츠       │ │
│ │ AI 요약: 가볍고 편한... │ │
│ │ 관련 키워드 #미니멀룩   │ │
│ └────────────────────────┘ │
│ ┌────────────────────────┐ │
│ │ 2. 와이드 데님          │ │
│ │ AI 요약: 10~20대 중심...│ │
│ └────────────────────────┘ │
│                            │
│ [내 트렌드][최신][저장][마이]│
└────────────────────────────┘
```

### 이 화면에서 중요한 구성

홈에서는 모든 정보를 길게 보여주기보다 **요약 중심**으로 보여주는 것이 좋습니다.

카드 하나에는 아래 요소만 들어가면 충분합니다.

```text
순위
트렌드 키워드
카테고리
짧은 AI 요약
관련 키워드 2~3개
저장 버튼
```

예시 카드:

```text
┌────────────────────────────┐
│ 1  여름 린넨 셔츠      ☆   │
│ 패션 · 오늘 상승            │
│ 가볍고 편한 여름 코디로     │
│ 10~20대 관심이 증가 중이에요│
│ #미니멀룩 #출근룩 #여름코디 │
└────────────────────────────┘
```

## 3-3. 관심사 기반 트렌드 탭

이 화면은 기획안의 핵심 기능입니다. 사용자가 선택한 관심 카테고리별로 트렌드를 확인하는 화면입니다.

```text
┌────────────────────────────┐
│ 내 관심사 트렌드            │
│                            │
│ [전체] [패션] [뷰티] [게임] │
│ [음식] [IT] [음악]          │
│                            │
│ 정렬: [인기순 ▼]  기간:[오늘]│
│                            │
│ ┌────────────────────────┐ │
│ │ 1  데님 온 데님         │ │
│ │ 패션 · 상승률 +32%       │ │
│ │ AI 요약 보기 >           │ │
│ └────────────────────────┘ │
│ ┌────────────────────────┐ │
│ │ 2  저속노화 식단        │ │
│ │ 음식 · 관련 검색 증가    │ │
│ │ AI 요약 보기 >           │ │
│ └────────────────────────┘ │
│ ┌────────────────────────┐ │
│ │ 3  모바일 신작 게임     │ │
│ │ 게임 · 커뮤니티 언급 증가│ │
│ │ AI 요약 보기 >           │ │
│ └────────────────────────┘ │
│                            │
│ [내 트렌드][최신][저장][마이]│
└────────────────────────────┘
```

### 설계 포인트

이 화면에서는 필터가 중요합니다.

```text
카테고리 필터: 패션 / 뷰티 / 게임 / 음식
기간 필터: 오늘 / 어제 / 주간
정렬 필터: 인기순 / 상승순 / 최신순
```

다만 UI가 복잡해지지 않도록 처음에는 상단에 칩 형태로 배치하는 것이 좋습니다.

```text
[전체] [패션] [뷰티] [게임] [음식]
```

기간 필터는 오른쪽 작은 드롭다운으로 처리합니다.

```text
오늘 ▼
```

## 3-4. 트렌드 상세 화면

트렌드 상세 화면은 이 앱의 차별화 포인트가 될 수 있습니다. 단순히 키워드만 보여주는 것이 아니라, **왜 뜨는지 / 관련 키워드는 무엇인지 / 어떤 카테고리와 연결되는지**를 보여주면 좋습니다.

```text
┌────────────────────────────┐
│ < 트렌드 상세          ☆    │
│                            │
│ 여름 린넨 셔츠              │
│ 패션 · 오늘 인기 상승        │
│                            │
│ ┌────────────────────────┐ │
│ │ AI 요약                 │ │
│ │ 최근 더운 날씨와 미니멀  │ │
│ │ 코디 선호가 겹치면서...  │ │
│ └────────────────────────┘ │
│                            │
│ 관련 키워드                 │
│ [미니멀룩] [출근룩] [여름코디]│
│                            │
│ 왜 주목받고 있나요?         │
│ ┌────────────────────────┐ │
│ │ 1. 계절성 이슈           │ │
│ │ 2. SNS 코디 콘텐츠 증가  │ │
│ │ 3. 쇼핑몰 노출 증가      │ │
│ └────────────────────────┘ │
│                            │
│ 관련 트렌드                 │
│ - 와이드 팬츠               │
│ - 반팔 니트                 │
│ - 여름 샌들                 │
└────────────────────────────┘
```

### 이 화면의 핵심 정보 구조

```text
1. 트렌드 키워드
2. 카테고리
3. AI 요약
4. 관련 키워드
5. 상승 이유
6. 관련 트렌드
7. 저장 버튼
```

여기서 “AI 요약”은 너무 길면 안 됩니다. 발표용으로는 아래 정도가 좋습니다.

```text
이 키워드는 최근 SNS와 쇼핑 플랫폼에서 여름 코디 관련 콘텐츠가 증가하면서 주목받고 있습니다.
```

## 3-5. 최신 이슈 탭

최신 이슈 탭은 보조 기능이므로, 개인화보다는 “전체 흐름 파악”에 초점을 둡니다.

```text
┌────────────────────────────┐
│ 최신 이슈                   │
│ 전체 사용자가 주목하는 흐름 │
│                            │
│ [오늘] [어제] [주간]        │
│                            │
│ ┌────────────────────────┐ │
│ │ 🔥 지금 많이 보는 이슈   │ │
│ │ 1. ○○ 페스티벌          │ │
│ │ 2. 신작 게임 공개        │ │
│ │ 3. 여름 장마 시작        │ │
│ └────────────────────────┘ │
│                            │
│ 카테고리별 인기 이슈        │
│ ┌────────────────────────┐ │
│ │ 패션                    │ │
│ │ #여름코디 #샌들 #반팔니트│ │
│ └────────────────────────┘ │
│ ┌────────────────────────┐ │
│ │ 게임                    │ │
│ │ #신작출시 #이벤트 #업데이트│ │
│ └────────────────────────┘ │
│                            │
│ [내 트렌드][최신][저장][마이]│
└────────────────────────────┘
```

### 관심사 탭과 최신 이슈 탭의 차이

| 화면      | 목적                  | 보여줄 정보                     |
| --------- | --------------------- | ------------------------------- |
| 내 트렌드 | 개인 관심사 기반 추천 | 내가 선택한 분야의 트렌드       |
| 최신 이슈 | 전체 흐름 파악        | 전체 사용자 기준 인기 이슈      |
| 상세 화면 | 이해와 분석           | AI 요약, 관련 키워드, 상승 이유 |

이 차이를 화면에서 명확히 보여줘야 합니다. 그렇지 않으면 두 탭이 비슷해 보일 수 있습니다.

# 4. 추천 화면 흐름

전체 사용자 흐름은 이렇게 정리하면 좋습니다.

```text
앱 실행
  ↓
로그인 / 회원가입
  ↓
관심 카테고리 선택
  ↓
메인 홈
  ↓
관심사 기반 트렌드 확인
  ↓
트렌드 상세 확인
  ↓
저장 또는 관련 키워드 탐색
```

최신 이슈 탭은 별도 보조 흐름입니다.

```text
메인 홈
  ↓
최신 이슈 탭
  ↓
오늘 / 어제 / 주간 필터 선택
  ↓
전체 인기 이슈 확인
  ↓
이슈 상세 확인
```

# 5. Figma 작업 기준 추천

현재 작업하신 UI가 둥근 카드, 연한 파스텔 톤, 귀여운 분위기에 가까웠기 때문에 이 방향을 유지한다면 아래처럼 통일하면 좋습니다.

## 카드 스타일

```text
배경: 연한 보라 / 연한 회색 / 흰색
모서리: 20~28px radius
그림자: 아주 약하게
텍스트: 제목은 굵게, 설명은 작고 흐리게
아이콘: 카테고리마다 작은 이모지 또는 라인 아이콘
```

## 트렌드 카드 구성

```text
┌────────────────────────────┐
│ 1  키워드 제목        ☆     │
│ 카테고리 · 상승률 · 기간     │
│ AI 요약 한 줄               │
│ #관련키워드 #관련키워드      │
└────────────────────────────┘
```

## 색상 역할

```text
메인 컬러: 연보라 / 블루 계열
강조 컬러: 진한 파랑
배경 컬러: 거의 흰색에 가까운 연회색
카드 컬러: 흰색 또는 아주 연한 보라
위험/상승 표시: 빨강보다는 오렌지나 핑크 정도
```

너무 많은 색을 쓰면 트렌드 앱이 아니라 쇼핑몰처럼 보일 수 있으니, 카테고리별 색상은 은은하게만 쓰는 것이 좋습니다.

# 6. 기능상세명세서로 확장하기 좋은 화면

이제 기획서를 기능상세명세서로 확장해야 한다면, 가장 먼저 아래 4개 화면을 상세화하는 것이 좋습니다.

```text
1. 관심 카테고리 선택 화면
2. 관심사 기반 트렌드 메인 화면
3. 트렌드 상세 화면
4. 최신 이슈 탭 화면
```

예를 들어 “관심 카테고리 선택 화면”은 이렇게 명세할 수 있습니다.

```text
기능명: 관심 카테고리 선택

목적:
사용자가 관심 있는 분야를 선택하여 이후 메인 화면에서 개인화된 트렌드를 제공받도록 한다.

사용자 동작:
- 사용자는 패션, 뷰티, 게임, 음식 등의 카테고리 중 3개 이상 선택한다.
- 선택 완료 버튼을 누른다.

처리 로직:
- 선택한 카테고리 ID 목록을 서버로 전송한다.
- 서버는 사용자 ID와 카테고리 ID를 매핑하여 저장한다.
- 저장 완료 후 메인 화면으로 이동한다.

데이터 저장:
- user_interest 테이블에 user_id, category_id를 저장한다.

예외 처리:
- 3개 미만 선택 시 “관심 분야를 3개 이상 선택해주세요.” 메시지를 표시한다.
- 서버 저장 실패 시 “관심사 저장에 실패했습니다. 다시 시도해주세요.” 메시지를 표시한다.
```

이런 방식으로 화면별 기능상세명세서를 이어가면 발표 자료와 구현 설계가 자연스럽게 연결됩니다.

# 7. 최종 추천 화면 구성안

가장 현실적인 MVP 화면 구성은 아래입니다.

```text
[필수]
1. 로그인 / 회원가입
2. 관심 카테고리 선택
3. 내 트렌드 홈
4. 카테고리별 트렌드 리스트
5. 트렌드 상세
6. 최신 이슈
7. 마이페이지

[선택]
8. 저장한 트렌드
9. 알림 설정
10. 검색 화면
```

현재 기획 수정안의 방향을 살리려면 화면 이름도 아래처럼 잡는 것이 좋습니다.

```text
메인 화면 → 내 트렌드
인기 검색어 → 관심사 트렌드
상세 페이지 → 트렌드 분석
전체 탭 → 최신 이슈
```

이렇게 바꾸면 서비스의 정체성이 더 선명해집니다.

핵심은 **“요즘 뭐가 유행하지?”**가 아니라,
**“내가 관심 있는 분야에서는 지금 뭐가 뜨고 있지?”**를 해결하는 앱으로 화면을 구성하는 것입니다.
